# База данных проекта (PostgreSQL)

В проекте используется PostgreSQL. База хранит фильмы, жанры и актёров, а также связи «многие-ко-многим».

---

## Таблицы

### `films` — Фильмы
Основные поля: `id` (PK), `title`, `release_year`, `runtime_min`, `description`.

### `genres` — Жанры
Поля: `id` (PK), `name` (уникальное название жанра).

### `film_genres` — Связь Фильм ↔ Жанр (many-to-many)
Поля: `film_id` (FK → `films.id`), `genre_id` (FK → `genres.id`).  
Составной первичный ключ: `(film_id, genre_id)`.

---

### `actors` — Актёры
Основные поля: `id` (PK), `full_name`, `birth_year`, `country`, `photo_url`.

### `actor_films` — Связь Актёр ↔ Фильм (many-to-many)
Поля: `actor_id` (FK → `actors.id`), `film_id` (FK → `films.id`).  
Составной первичный ключ: `(actor_id, film_id)`.

---

## Связи и целостность данных

- Один фильм может иметь несколько жанров, и один жанр может относиться к нескольким фильмам → `film_genres`
- Один актёр может сниматься в нескольких фильмах, и в одном фильме может быть много актёров → `actor_films`
- Внешние ключи (FK) защищают целостность: нельзя удалить запись, если на неё есть ссылки (если не настроен CASCADE).
